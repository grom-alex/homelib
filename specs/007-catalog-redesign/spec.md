# Feature Specification: Редизайн каталога в стиле MyHomeLib

**Feature Branch**: `007-catalog-redesign`
**Created**: 2026-02-23
**Status**: Draft
**Input**: User description: "нужно переделать дизайн основного сайта используя за основу дизайн из макета docs/design/myhomelib-catalog.jsx"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Трёхпанельный интерфейс каталога (Priority: P1)

Пользователь открывает каталог и видит интерфейс, разделённый на три зоны: левая навигационная панель (авторы/серии/жанры/поиск), центральная таблица книг и нижняя панель с детальной информацией о выбранной книге. Левая панель переключается вкладками в хедере. При выборе элемента в левой панели (автор, серия, жанр) таблица книг фильтруется соответственно. При клике на строку в таблице — внизу отображаются детали книги.

**Why this priority**: Это ядро нового дизайна — без трёхпанельной раскладки все остальные фичи не имеют смысла. Заменяет текущий карточный layout на табличный с навигацией.

**Independent Test**: Открыть каталог, выбрать автора в левой панели, убедиться что таблица показывает его книги, кликнуть на книгу — увидеть детали внизу.

**Acceptance Scenarios**:

1. **Given** пользователь авторизован, **When** открывает каталог, **Then** видит трёхпанельный layout: левая панель навигации, таблица книг справа сверху, панель деталей справа снизу
2. **Given** каталог открыт, **When** пользователь выбирает автора в левой панели, **Then** таблица книг фильтруется и показывает только книги этого автора
3. **Given** таблица содержит книги, **When** пользователь кликает на строку книги, **Then** в нижней панели отображаются детальные сведения о книге (название, автор, аннотация, метаданные, кнопки действий)
4. **Given** левая панель показывает вкладку «Авторы», **When** пользователь переключает вкладку на «Серии», **Then** левая панель показывает список серий, таблица книг очищается до выбора серии

---

### User Story 2 - Табличное представление книг с сортировкой (Priority: P1)

Книги отображаются в табличном виде с колонками: Название, Автор, Серия, Жанр, Размер. Пользователь может сортировать таблицу по любой колонке, кликнув на заголовок. Повторный клик на ту же колонку переключает направление сортировки (asc/desc).

**Why this priority**: Таблица — основной способ просмотра книг в новом дизайне, без неё каталог не функционален.

**Independent Test**: Открыть каталог с книгами, кликнуть на заголовок колонки «Автор», убедиться что книги отсортированы по автору.

**Acceptance Scenarios**:

1. **Given** таблица показывает список книг, **When** пользователь кликает на заголовок колонки «Название», **Then** книги сортируются по названию в алфавитном порядке, заголовок колонки визуально выделяется
2. **Given** таблица отсортирована по названию (asc), **When** пользователь снова кликает на «Название», **Then** направление сортировки меняется на desc, индикатор стрелки переворачивается
3. **Given** таблица отсортирована по названию, **When** пользователь кликает на другую колонку «Автор», **Then** сортировка переключается на «Автор» (asc), предыдущая колонка теряет выделение

---

### User Story 3 - Навигационные вкладки: Авторы, Серии, Жанры, Поиск (Priority: P1)

В верхнем хедере расположены четыре вкладки навигации, которые переключают содержимое левой панели:
- **Авторы** — список авторов с поиском и счётчиком книг
- **Серии** — список серий с поиском и счётчиком книг
- **Жанры** — древовидная структура жанров с раскрывающимися категориями
- **Поиск** — форма с полями для расширенного поиска (название, автор, серия, жанр, формат)

**Why this priority**: Вкладки — единственный способ навигации по каталогу в новом дизайне.

**Independent Test**: Переключиться между всеми четырьмя вкладками и убедиться, что содержимое левой панели меняется корректно.

**Acceptance Scenarios**:

1. **Given** открыта вкладка «Авторы», **When** пользователь вводит текст в поле поиска, **Then** список авторов фильтруется в реальном времени
2. **Given** открыта вкладка «Жанры», **When** пользователь кликает на категорию (например, «Фантастика»), **Then** раскрывается список поджанров; повторный клик — сворачивает
3. **Given** открыта вкладка «Жанры», **When** пользователь кликает на конкретный поджанр, **Then** таблица книг показывает книги этого поджанра
4. **Given** открыта вкладка «Поиск», **When** пользователь заполняет критерии и нажимает «Найти», **Then** таблица показывает результаты поиска

---

### User Story 4 - Цветовые схемы каталога (Priority: P2)

Каталог поддерживает четыре цветовые схемы: Light (текущая тема Vuetify), Dark (тёмная тема из макета MyHomeLib с золотистым акцентом #d4a017), Sepia (тёплые тона для комфортного чтения) и Night (минимальная яркость для тёмных помещений). Архитектура тем расширяемая — добавление новых схем в будущем не требует переработки существующего кода. Элементы интерфейса имеют стилистику «IDE/десктопного приложения»: компактные отступы, моноширинный шрифт для размеров файлов, subtle borders и hover-эффекты.

Читалка наследует тему каталога по умолчанию, но может быть переключена на другую тему независимо. У пользователя есть возможность вернуть читалку к теме каталога.

**Why this priority**: Визуальный стиль определяет восприятие приложения, а поддержка нескольких тем повышает комфорт пользователей. Функционально каталог работает и с одной темой.

**Independent Test**: Переключить тему каталога между всеми четырьмя схемами и убедиться, что цвета корректно применяются ко всем элементам интерфейса.

**Acceptance Scenarios**:

1. **Given** пользователь открывает каталог впервые, **Then** применяется тема по умолчанию (Light)
2. **Given** пользователь переключает тему на Dark, **Then** фон основной области тёмный, текст светлый, акцентные элементы (активная вкладка, выделенные строки) используют золотистый цвет
3. **Given** пользователь наводит курсор на строку в таблице, **Then** строка подсвечивается фоном, соответствующим текущей теме (hover-эффект)
4. **Given** пользователь выбирает тему Sepia, **Then** все элементы интерфейса перекрашиваются в тёплые тона
5. **Given** пользователь выбрал тему Dark для каталога, **When** открывает книгу в читалке, **Then** читалка открывается в теме Dark (наследование)
6. **Given** пользователь сменил тему читалки на Night, **When** закрывает читалку и открывает снова, **Then** читалка открывается в Night (независимый выбор сохраняется)
7. **Given** пользователь в читалке выбрал отличную от каталога тему, **When** выбирает «Сбросить к теме каталога», **Then** тема читалки возвращается к текущей теме каталога

---

### User Story 5 - Изменяемые размеры панелей (Priority: P3)

Пользователь может перетаскивать разделители (resizers) между панелями, чтобы настроить ширину левой панели и высоту верхней/нижней области (таблица/детали). Границы изменения размера ограничены минимальными/максимальными значениями.

**Why this priority**: Удобная функция, но не блокирует базовое использование каталога.

**Independent Test**: Перетащить вертикальный разделитель между левой панелью и таблицей, убедиться что ширина панелей меняется.

**Acceptance Scenarios**:

1. **Given** каталог открыт, **When** пользователь зажимает и тянет вертикальный разделитель, **Then** ширина левой панели увеличивается/уменьшается, таблица занимает оставшееся пространство
2. **Given** каталог открыт, **When** пользователь зажимает и тянет горизонтальный разделитель, **Then** высота таблицы и панели деталей меняется пропорционально
3. **Given** пользователь тянет разделитель за минимальный предел, **Then** размер панели не уменьшается ниже минимума

---

### User Story 6 - Панель деталей книги (Priority: P2)

При выборе книги в таблице нижняя панель показывает: название, автора, серию (с номером), жанр, год, формат, размер, язык, аннотацию, а также кнопки «Читать» и «Скачать». Кнопка «Читать» доступна для поддерживаемых форматов (fb2).

**Why this priority**: Детальная информация необходима для принятия решения о чтении/скачивании книги.

**Independent Test**: Выбрать книгу в таблице и проверить, что нижняя панель показывает все поля метаданных и кнопки действий.

**Acceptance Scenarios**:

1. **Given** книга выбрана, **Then** в панели деталей отображаются: название, автор, серия (#номер), жанр, год, формат, размер, язык, аннотация
2. **Given** выбрана книга формата fb2, **Then** кнопка «Читать» активна и ведёт на страницу чтения
3. **Given** книга выбрана, **When** пользователь нажимает «Скачать», **Then** начинается скачивание файла книги
4. **Given** ни одна книга не выбрана, **Then** панель деталей показывает placeholder-текст «Выберите книгу для просмотра подробной информации»

---

### User Story 7 - Хедер с пользовательским меню (Priority: P2)

В хедере справа отображается аватар пользователя с именем и выпадающее меню с пунктами: Мой профиль, Настройки, Мои коллекции, Загрузить книги, Выйти. Также в хедере отображается общее количество книг в библиотеке.

**Why this priority**: Пользовательское меню необходимо для навигации по аккаунту, но не блокирует основной workflow каталога.

**Independent Test**: Кликнуть на аватар в хедере, убедиться что появляется дропдаун с пунктами меню.

**Acceptance Scenarios**:

1. **Given** пользователь авторизован, **Then** в хедере отображается его аватар (инициалы) и имя
2. **Given** пользователь кликает на аватар, **Then** появляется дропдаун-меню с пунктами: Мой профиль, Настройки, Мои коллекции, Загрузить книги, Выйти
3. **Given** дропдаун открыт, **When** пользователь кликает вне меню, **Then** меню закрывается
4. **Given** пользователь кликает «Выйти», **Then** выполняется выход из системы

---

### User Story 8 - Статус-бар (Priority: P3)

Внизу экрана отображается статус-бар с контекстной информацией: выбранный фильтр (например, «Автор: Азимов, Айзек») слева и количество показанных книг справа.

**Why this priority**: Информативный элемент, улучшающий UX, но не критичный для функциональности.

**Independent Test**: Выбрать автора и проверить, что статус-бар показывает имя автора и количество книг.

**Acceptance Scenarios**:

1. **Given** пользователь выбрал автора, **Then** в статус-баре слева: «Автор: [имя]», справа: «Показано книг: [N]»
2. **Given** ничего не выбрано, **Then** статус-бар слева: «Готов»

---

### Edge Cases

- Что происходит, когда список авторов/серий пуст (новая библиотека без книг)? — Показывается сообщение «Ничего не найдено»
- Что происходит, когда поиск в левой панели не даёт результатов? — Показывается пустое состояние с сообщением
- Как ведёт себя интерфейс при очень длинных названиях книг/авторов? — Текст обрезается с ellipsis
- Что происходит при переключении вкладки, если была выбрана книга? — Выделение книги сбрасывается, панель деталей показывает placeholder
- Как интерфейс адаптируется к маленьким экранам (< 1024px)? — Минимальная ширина панелей ограничена; при очень узком экране можно скроллить горизонтально
- Что если у книги нет аннотации? — Раздел «Аннотация» не отображается или показывает «Аннотация отсутствует»

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: Система MUST отображать трёхпанельный интерфейс: левая навигационная панель, таблица книг (верх-право), панель деталей (низ-право)
- **FR-002**: Система MUST поддерживать четыре режима навигации через вкладки в хедере: Авторы, Серии, Жанры, Поиск
- **FR-003**: Вкладка «Авторы» MUST показывать прокручиваемый список авторов с полем поиска и счётчиком книг у каждого автора
- **FR-004**: Вкладка «Серии» MUST показывать список серий с полем поиска и счётчиком книг
- **FR-005**: Вкладка «Жанры» MUST показывать древовидную структуру жанров с раскрывающимися категориями
- **FR-006**: Вкладка «Поиск» MUST предоставлять форму расширенного поиска с полями: название, автор, серия, жанр (dropdown), формат (dropdown) и кнопками «Найти»/«Очистить»
- **FR-007**: Таблица книг MUST отображать колонки: Название, Автор, Серия, Жанр, Размер
- **FR-008**: Таблица книг MUST поддерживать сортировку по любой колонке с визуальным индикатором направления (asc/desc)
- **FR-009**: При выборе книги в таблице панель деталей MUST отображать: название, автор, серия (#номер), жанр, год, формат, размер, язык, аннотация
- **FR-010**: Панель деталей MUST содержать кнопки «Читать» (для fb2) и «Скачать»
- **FR-011**: Пользователь MUST иметь возможность изменять ширину левой панели и высоту разделения таблица/детали перетаскиванием разделителей
- **FR-012**: Хедер MUST содержать логотип, навигационные вкладки, счётчик книг и пользовательское меню с аватаром
- **FR-013**: Пользовательское дропдаун-меню MUST содержать пункты: Мой профиль, Настройки, Мои коллекции, Загрузить книги, Выйти
- **FR-014**: Статус-бар MUST отображать текущий контекст (выбранный фильтр) и количество показанных книг
- **FR-015**: Система MUST поддерживать четыре цветовые схемы: Light (текущая Vuetify), Dark (макет MyHomeLib), Sepia (тёплые тона), Night (минимальная яркость)
- **FR-016**: При выборе элемента в левой панели (автор/серия/жанр) таблица книг MUST фильтроваться и показывать только соответствующие книги
- **FR-017**: Текст в ячейках таблицы MUST обрезаться с многоточием при переполнении
- **FR-018**: Архитектура тем MUST быть расширяемой — добавление новой цветовой схемы не должно требовать изменений в существующих компонентах
- **FR-019**: Пользовательское дропдаун-меню MUST содержать быстрый переключатель цветовой схемы каталога
- **FR-020**: Пункт «Настройки» в меню MUST открывать диалог с полным управлением темами (выбор темы каталога, выбор темы читалки)
- **FR-021**: Читалка MUST наследовать тему каталога по умолчанию при первом открытии
- **FR-022**: Пользователь MUST иметь возможность переключить тему читалки независимо от темы каталога
- **FR-023**: В настройках темы читалки MUST быть опция «Сбросить к теме каталога» для возврата к наследованию
- **FR-024**: Выбранные темы (каталога и читалки) MUST сохраняться в пользовательских настройках на сервере

### Key Entities

- **Левая панель (Navigation Panel)**: переключаемая панель с содержимым, определяемым активной вкладкой; содержит списки с поиском или древовидную структуру
- **Таблица книг (Book Table)**: табличное представление книг с фиксированными колонками и сортировкой; данные зависят от выбора в левой панели
- **Панель деталей (Detail Panel)**: отображает метаданные и аннотацию выбранной книги с кнопками действий
- **Разделители (Resizers)**: перетаскиваемые элементы для изменения размеров панелей
- **Цветовая схема (Theme)**: набор цветовых переменных (фон, текст, акценты, бордеры), определяющий внешний вид интерфейса; хранится отдельно для каталога и читалки

## Assumptions

- Текущие API-эндпоинты (GET /api/books, GET /api/authors, GET /api/genres, GET /api/series) предоставляют достаточно данных для нового UI
- Аннотации книг уже есть в базе данных (или поле annotation доступно через API)
- Рейтинг книг пока не реализован в бэкенде — в первой версии не показывается
- Текущая страница деталей книги (BookView) и читалка (ReaderView) остаются без изменений (кроме интеграции с темой каталога)
- Маршруты /authors, /genres, /series как отдельные страницы заменяются на вкладки в едином каталоге
- Пункты «Мой профиль», «Мои коллекции», «Загрузить книги» в меню пока показывают заглушку (функционал будущих фич)
- Существующий эндпоинт PUT /me/settings с JSONB-полем используется для хранения выбранных тем (каталога и читалки)
- Тема по умолчанию — Light (текущая тема Vuetify)

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Пользователь может найти и открыть детали книги конкретного автора за 3 клика (вкладка -> автор -> книга)
- **SC-002**: Переключение между вкладками навигации происходит мгновенно (без задержки загрузки)
- **SC-003**: Таблица книг отображает до 100 записей без заметных задержек прокрутки
- **SC-004**: Визуальное соответствие макету из docs/design/myhomelib-catalog.jsx при теме Dark: золотистые акценты, трёхпанельный layout, табличный вид книг; все четыре темы корректно применяются ко всем элементам
- **SC-005**: Все существующие функции каталога (фильтрация, поиск, навигация к книге, чтение fb2) сохраняют работоспособность после редизайна
- **SC-006**: Интерфейс корректно отображается на экранах шириной от 1280px

### Session 2026-02-23

- Q: Какие цветовые схемы должны поддерживаться? → A: Light (текущая Vuetify) + Dark (макет MyHomeLib) + Sepia + Night. Архитектура должна быть расширяемой для добавления новых схем в будущем.
- Q: Как связаны темы каталога и читалки? → A: Читалка наследует тему каталога по умолчанию, но может быть переключена отдельно с возможностью возврата к теме каталога.
- Q: Где расположен переключатель тем? → A: Оба варианта: быстрый переключатель в пользовательском дропдаун-меню и полный диалог настроек через пункт «Настройки».
