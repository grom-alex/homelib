# Feature Specification: Древовидная структура жанров

**Feature Branch**: `008-genre-tree`
**Created**: 2026-03-01
**Status**: Draft
**Input**: Внедрение древовидной иерархии жанров из файла `docs/genres_all.glst` с загрузкой в каталог и навигацией по дереву жанров в UI.

## Анализ файла жанров

### Формат файла `genres_all.glst`

Файл содержит 448 жанровых записей в древовидной структуре:
- **Уровень 0** (корневые категории, 2 цифры: `0.X`): 27 записей
- **Уровень 1** (подкатегории, 3 цифры: `0.X.Y`): 343 записи
- **Уровень 2** (под-подкатегории, 4 цифры: `0.X.Y.Z`): 67 записей
- **Уровень 3** (4-й уровень, 5 цифр: `0.X.Y.Z.W`): 11 записей

Формат строки: `<уровень> <код>;<имя>` (комментарии начинаются с `#`).

Один и тот же код жанра присутствует в нескольких позициях дерева:

### Записи без кода (1 шт.)

- `0.0 Неотсортированное` — нет разделителя `;`, код отсутствует

### Правила уникальности кодов

- **Корневой уровень (уровень 0)**: дубликаты кодов **запрещены**. Каждая корневая категория ДОЛЖНА иметь уникальный код. При обнаружении дубликата на корневом уровне — логируется ошибка.
- **Уровни 1, 2, 3 (дочерние)**: дубликаты кодов **допускаются**. Один и тот же INPX-код может присутствовать в нескольких позициях дерева (например, `home_cooking` входит и в «Домоводство» 0.14, и в «Дом. Быт. Досуг» 0.24). Это означает, что поджанр может принадлежать нескольким родительским жанрам.

При маппинге книг из INPX:
- Книга с кодом-дубликатом привязывается ко **всем** позициям дерева, где этот код присутствует
- Записи без кода получают автоматически сгенерированный код на основе позиции (например, `_root_0`)

## Clarifications

### Session 2026-03-01

- Q: Каким способом загружается справочник жанров (.glst)? → A: Файл поставляется с приложением, автозагрузка при старте если дерево отсутствует или устарело. Дополнительно — CLI-команда для ручной перезагрузки (для отладки).
- Q: Нужен ли автоматический ремаппинг существующих книг при загрузке/обновлении дерева жанров? → A: Да, автоматический ремаппинг всех книг при каждой загрузке/обновлении дерева.
- Q: Как обрабатывать синтаксические ошибки в файле .glst? → A: Пропускать невалидные строки, логировать предупреждение, загружать остальные корректные записи.
- Q: Куда попадают книги без жанра или с неизвестным кодом жанра? → A: В категорию «Неотсортированное» (позиция 0.0). Это относится как к книгам с пустым полем жанра, так и к книгам с кодом, отсутствующим в справочнике.
- Q: Где допускаются дубликаты кодов жанров? → A: Дубликаты кодов допускаются только на уровнях глубже корневого (уровни 1, 2, 3) — поджанр может входить в несколько родительских жанров. На корневом уровне (уровень 0) дубликаты кодов запрещены.
- Q: Нужна ли новая колонка в таблице books для хранения оригинальных INPX-кодов жанров? → A: Нет. Оригинальные INPX-коды уже хранятся в поле `genres.code`. Ремаппинг выполняется через JOIN `book_genres → genres.code` → поиск узлов дерева по коду. Дополнительная колонка не требуется.

## User Scenarios & Testing

### User Story 1 — Просмотр дерева жанров в каталоге (Priority: P1)

Пользователь открывает вкладку «Жанры» в каталоге и видит древовидную иерархию жанров вместо плоского списка. Корневые категории (Фантастика, Детективы, Проза и т.д.) отображаются как раскрываемые узлы. При раскрытии категории видны подкатегории, которые также могут содержать вложенные уровни. Рядом с каждым жанром отображается количество книг.

**Why this priority**: Это основной сценарий использования — навигация по жанрам является одним из главных способов поиска книг в библиотеке. Древовидная структура позволяет быстро ориентироваться в 448 жанрах.

**Independent Test**: Загрузить файл жанров, открыть вкладку «Жанры», убедиться что дерево отображается корректно с раскрытием/сворачиванием узлов.

**Acceptance Scenarios**:

1. **Given** файл жанров загружен в систему, **When** пользователь открывает вкладку «Жанры», **Then** отображается список из 27 корневых категорий в свёрнутом виде с количеством книг.
2. **Given** пользователь видит список корневых категорий, **When** пользователь раскрывает категорию «Фантастика», **Then** отображаются 32 подкатегории (Альтернативная история, Боевая фантастика и т.д.) с количеством книг у каждой.
3. **Given** пользователь раскрыл категорию «Наука, Образование», **When** пользователь раскрывает подкатегорию «Физика», **Then** отображаются 13 под-подкатегорий (Акустика, Квантовая механика и т.д.).
4. **Given** пользователь видит дерево жанров, **When** пользователь нажимает на имя жанра (не на стрелку раскрытия), **Then** в правой панели отображается список книг этого жанра, включая книги из всех дочерних жанров.
5. **Given** пользователь нажал на корневую категорию «Фантастика», **Then** отображаются все книги, принадлежащие жанру «Фантастика» и всем его подкатегориям.

### User Story 2 — Автозагрузка справочника жанров при старте (Priority: P1)

Файл `.glst` поставляется вместе с приложением. При запуске система проверяет, загружен ли справочник жанров и актуален ли он. Если дерево отсутствует или устарело — система автоматически парсит встроенный файл, строит иерархическую структуру и сохраняет в базу данных. При обновлении приложения с новой версией файла дерево обновляется: новые жанры добавляются, удалённые — помечаются неактивными, изменённые — обновляются.

**Why this priority**: Без загрузки справочника жанров остальные сценарии невозможны. Автозагрузка при старте обеспечивает готовность системы без ручного вмешательства администратора.

**Independent Test**: Запустить приложение с пустой БД, проверить что в базе автоматически появились записи жанров с корректной иерархией.

**Acceptance Scenarios**:

1. **Given** система запущена впервые (дерево жанров отсутствует в БД), **When** приложение стартует, **Then** файл `.glst` автоматически загружается и в базе создаются записи жанров с корректными родительскими связями.
2. **Given** в файле есть записи без кода (`0.0 Неотсортированное`), **When** файл загружается, **Then** для таких записей генерируется код на основе позиции и они корректно включаются в дерево.
3. **Given** в файле есть дубликаты кодов (`home_cooking` в позициях 0.14.1 и 0.24.4), **When** файл загружается, **Then** оба узла дерева создаются с одним кодом — каждый в своей позиции иерархии.
4. **Given** справочник жанров уже загружен, **When** приложение перезапускается с обновлённым файлом `.glst`, **Then** новые жанры добавляются, существующие обновляются, удалённые из файла жанры остаются в базе (не удаляются — у них могут быть привязанные книги).
5. **Given** файл загружен успешно, **When** система получает данные об INPX-импорте с кодом жанра `sf_action`, **Then** книга привязывается к узлу дерева «Боевая фантастика» (позиция 0.1.2).
6. **Given** в базе уже есть книги с плоскими кодами жанров (импортированные ранее), **When** дерево жанров загружается впервые, **Then** все существующие книги автоматически перепривязываются к соответствующим узлам дерева по их INPX-кодам.
7. **Given** разработчику нужно перезагрузить справочник вручную, **When** он выполняет CLI-команду перезагрузки, **Then** дерево жанров перезагружается из файла и ремаппинг книг выполняется повторно.

### User Story 3 — Фильтрация книг по жанру в дереве (Priority: P2)

Пользователь выбирает жанр на любом уровне дерева и видит список книг. При выборе родительского жанра отображаются книги этого жанра и всех его потомков. Пользователь может уточнять выбор, спускаясь по дереву глубже.

**Why this priority**: Фильтрация по жанру — ключевая функция каталога, но она работает и в текущей реализации (плоский список). Улучшение — каскадная фильтрация по иерархии.

**Independent Test**: Выбрать корневой жанр, проверить что отображаются книги всех подкатегорий; затем выбрать подкатегорию — должны отобразиться только книги этой подкатегории и её потомков.

**Acceptance Scenarios**:

1. **Given** пользователь на вкладке «Жанры», **When** пользователь выбирает «Наука, Образование» (корневой), **Then** отображаются все книги из этой категории и всех вложенных подкатегорий (История, Психология, Физика→Акустика и т.д.).
2. **Given** пользователь на вкладке «Жанры», **When** пользователь выбирает «Физика» (уровень 1), **Then** отображаются книги только по Физике и всем её подкатегориям (Акустика, Квантовая механика и т.д.), но не из других подкатегорий «Наука, Образование».
3. **Given** пользователь выбрал жанр с дубликатом кода (`home_cooking`), **When** книга привязана к этому коду, **Then** книга отображается при выборе любой из позиций дерева, содержащих этот код.

### User Story 4 — Поиск жанров в дереве (Priority: P2)

Пользователь может искать жанр по названию прямо на вкладке «Жанры». При вводе текста дерево фильтруется: отображаются только жанры, чьё название содержит введённый текст, вместе с их родительскими узлами (для сохранения контекста иерархии).

**Why this priority**: При 448 жанрах найти нужный через раскрытие дерева может быть затруднительно. Поиск значительно ускоряет навигацию.

**Independent Test**: Ввести «физика» в поле поиска, убедиться что отображаются все жанры со словом «физика» в названии с их родительской цепочкой.

**Acceptance Scenarios**:

1. **Given** пользователь на вкладке «Жанры», **When** пользователь вводит «физика» в поле поиска, **Then** отображаются: «Наука, Образование → Физика», «Наука, Образование → Физика → Математическая физика», «Наука, Образование → Биофизика» и другие совпадения.
2. **Given** пользователь ввёл поисковый запрос «xyz123», **When** совпадений нет, **Then** отображается сообщение «Жанры не найдены».
3. **Given** пользователь нашёл жанр через поиск, **When** пользователь очищает поле поиска, **Then** дерево возвращается в исходное состояние.

### User Story 5 — Использование жанров в фильтрах поиска книг (Priority: P3)

На вкладке «Поиск» пользователь может выбрать жанр из выпадающего древовидного списка для уточнения результатов поиска. Выбор работает по тому же принципу — выбор родительского жанра включает все дочерние.

**Why this priority**: Расширяет уже существующую функциональность поиска. Текущий фильтр по жанру (`genre_id`) работает, но привязан к плоской структуре.

**Independent Test**: Открыть вкладку «Поиск», выбрать жанр из выпадающего дерева, выполнить поиск и проверить что результаты ограничены выбранным жанром и его потомками.

**Acceptance Scenarios**:

1. **Given** пользователь на вкладке «Поиск», **When** пользователь нажимает на фильтр по жанру, **Then** открывается выпадающий древовидный список жанров с возможностью раскрытия категорий.
2. **Given** пользователь выбрал «Детективы и Триллеры» в фильтре, **When** пользователь выполняет текстовый поиск, **Then** результаты ограничены книгами из жанра «Детективы и Триллеры» и всех его подкатегорий.

### Edge Cases

- **Синтаксические ошибки в `.glst`**: Невалидные строки пропускаются с логированием предупреждения. Остальные корректные записи загружаются нормально.
- **Книги без жанра или с неизвестным кодом**: Книги с пустым полем жанра, а также книги с кодами, отсутствующими в дереве, привязываются к категории «Неотсортированное» (позиция 0.0).
- **Жанры с 0 книг**: Отображаются в дереве (не скрываются), чтобы пользователь видел полную структуру справочника.
- **Пустой файл `.glst`**: Загрузка пропускается, логируется предупреждение. Существующее дерево в БД не затрагивается.
- **Дублирующиеся позиции**: Если в файле две строки с одинаковой позицией — загружается последняя (перезаписывает первую), логируется предупреждение.
- **Дочерние узлы без родителя**: Если в файле есть запись уровня 1+ (например, `0.27.0`), но родительский узел (например, `0.27`) отсутствует — дочерние узлы пропускаются с логированием предупреждения.

## Requirements

### Functional Requirements

- **FR-001**: Система ДОЛЖНА парсить файлы формата `.glst`, поддерживая иерархию до 4 уровней вложенности (корневой + 3 дочерних).
- **FR-002**: Система ДОЛЖНА строить древовидную структуру жанров на основе числового пути (позиции) каждого жанра, где первые две цифры определяют родителя корневого уровня, три цифры — уровень 1 и так далее.
- **FR-003**: Система ДОЛЖНА корректно обрабатывать дубликаты кодов на дочерних уровнях (1, 2, 3): один INPX-код может быть привязан к нескольким позициям дерева. На корневом уровне (уровень 0) дубликаты кодов запрещены.
- **FR-004**: Система ДОЛЖНА обрабатывать записи без кода жанра (отсутствует разделитель `;`), генерируя код автоматически.
- **FR-005**: Система ДОЛЖНА игнорировать строки-комментарии (начинающиеся с `#`) и пустые строки при парсинге.
- **FR-006**: Система ДОЛЖНА хранить для каждого жанра: код, человекочитаемое название, позицию в дереве, ссылку на родительский жанр.
- **FR-007**: При выборе жанра в каталоге система ДОЛЖНА отображать книги данного жанра и всех его потомков (каскадная фильтрация).
- **FR-008**: Вкладка «Жанры» ДОЛЖНА отображать жанры в виде раскрываемого дерева с количеством книг у каждого жанра (включая книги потомков).
- **FR-009**: Вкладка «Жанры» ДОЛЖНА предоставлять поле поиска для фильтрации жанров по названию с сохранением родительской цепочки для найденных жанров.
- **FR-010**: Вкладка «Поиск» ДОЛЖНА предоставлять фильтр по жанру в виде выпадающего древовидного списка.
- **FR-011**: При импорте книг из INPX система ДОЛЖНА маппить INPX-коды жанров на узлы дерева жанров.
- **FR-012**: Книги без указанного жанра, а также книги с INPX-кодами, отсутствующими в дереве жанров, ДОЛЖНЫ помещаться в категорию «Неотсортированное» (позиция 0.0).
- **FR-013**: Обновление справочника жанров ДОЛЖНО быть идемпотентным: повторная загрузка того же файла не должна создавать дубликаты или терять привязки книг.
- **FR-014**: Счётчик книг у родительского жанра ДОЛЖЕН включать книги из всех дочерних жанров (рекурсивный подсчёт).
- **FR-015**: При загрузке или обновлении дерева жанров система ДОЛЖНА автоматически перепривязать все существующие книги к узлам дерева по их INPX-кодам.
- **FR-016**: Система ДОЛЖНА предоставлять CLI-команду и HTTP admin endpoint (`POST /api/admin/genres/reload`) для ручной перезагрузки справочника жанров из файла `.glst` (для отладки и администрирования).
- **FR-017**: При парсинге файла `.glst` система ДОЛЖНА пропускать невалидные строки (с логированием предупреждения) и продолжать загрузку остальных корректных записей.

### Key Entities

- **Genre (Жанр)**: Узел дерева жанров. Содержит: код (из INPX), человекочитаемое название, позицию в дереве (числовой путь), ссылку на родительский жанр, признак активности. Один код может соответствовать нескольким узлам дерева.
- **Genre Tree (Дерево жанров)**: Иерархическая структура из 448 жанров, организованная в 4 уровня вложенности. Загружается из файла `.glst`. Корневой уровень содержит 27 категорий.
- **Book-Genre Mapping (Связь книга-жанр)**: Связь M:N между книгами и жанрами. Одна книга может принадлежать нескольким жанрам. При маппинге через INPX-код книга привязывается ко всем узлам дерева, содержащим этот код.

## Success Criteria

### Measurable Outcomes

- **SC-001**: Пользователь может найти нужный жанр и увидеть список его книг за 3 клика или менее (раскрытие категории → раскрытие подкатегории → выбор жанра).
- **SC-002**: Все 448 жанров из файла `genres_all.glst` загружены и отображаются в корректной иерархии.
- **SC-003**: При выборе корневой категории пользователь видит книги из всех её подкатегорий (каскадная фильтрация работает на всех уровнях).
- **SC-004**: Поиск жанра по части названия возвращает результаты менее чем за 200 мс (без задержки, воспринимаемой пользователем).
- **SC-005**: 100% книг привязаны к дереву жанров: книги с известными кодами — к соответствующим узлам, книги без жанра или с неизвестным кодом — к «Неотсортированное».
- **SC-006**: Повторная загрузка файла `.glst` не создаёт дубликатов жанров и не теряет существующие привязки книг.

## Assumptions

- Файл `.glst` является единственным источником истины для структуры жанров. INPX-файлы содержат только коды жанров без иерархии.
- Файл `.glst` поставляется встроенным в приложение и загружается автоматически при старте, если дерево жанров отсутствует или устарело. Дополнительно доступна CLI-команда для ручной перезагрузки (для отладки и разработки).
- Максимальная глубина дерева — 4 уровня (корневой + 3). Если в будущем появятся более глубокие уровни, система должна их поддерживать без изменений логики.
- Жанры без книг отображаются в дереве (не скрываются), чтобы пользователь видел полную структуру.
- Порядок отображения жанров определяется их позицией в файле `.glst` (числовой путь).
- Книги без жанра и книги с немаппированными INPX-кодами привязываются к «Неотсортированное», а не игнорируются.
- Дубликаты кодов жанров допускаются только на дочерних уровнях (1, 2, 3). На корневом уровне (0) каждый код уникален.
- Оригинальные INPX-коды жанров книг восстанавливаются из существующей связи `book_genres → genres.code`. Новая колонка в таблице `books` для хранения сырых кодов не требуется.

## Dependencies

- Существующая таблица `genres` в БД (миграция 001) — потребуется модификация для поддержки позиции в дереве.
- Существующий API-эндпоинт `GET /api/genres` — потребуется расширение для возврата дерева.
- Существующий компонент `GenresTab.vue` — потребуется переработка для отображения дерева.
- Процесс импорта INPX (`worker`) — потребуется обновление маппинга жанров.
